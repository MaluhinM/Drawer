Краткая инструкция к использованию мини-игры.
Основная программа запускается файлом 'Minigame -2024.exe'.
Управление указано в файле 'Управление.txt'.
При желании посмотреть на программу с командной строкой, следует запустить файл 'console.exe'.

Детали:
Файл 'Minigame -2024.exe' запускает основной алгоритм программы, который использует следующие файлы\папки:
-> <figures.py> хранит в себе алгоритмы 'фигур', которые можно вызывать определенными клавишами или сочетаниями клавиш.
-> <data> - основная папка с информацией для работы программы, которая также является папкой для настройки программы.

Также в файлах есть программа <CreateNewFigure.exe> для упрощения создания 'фигур' (алгоритмов).
Если Вы не знаете языка программирования Python и не понимаете сути создания фигур - лучше не используйте эту программу.

Для установки пакетов данных (дата-паков) используется программа <InstallDataPack.exe>. Если путь <data\DataPacks> отсутствует, то при запуске предложенной программы та создаст нужную папку. Все пакеты данных начинаются либо с <dp_>, либо с <datapack_> и загружаются в папку <DataPacks>.
В случае запуска программы с имеющейся папкой, та начнет поиск пакетов данных и предложит установить имеющиеся (в случае отсутствия пакетов программа предложит завершить работу). При выборе пакета для установки программа сначала покажет его содержимое, а затем уже предложит его установить (в случае отказа от установки Вы вернетесь к списку пакетов). При соглашении на установку пакета программа начнет его обрабатывать (обработка обычно занимает менее секунды, но это время зависит от 'мощности' Вашего устройства). При обработке пакета в консоль будет выводится информация, которая содержится в обрабатываемом пакете (в случае возникновения несостыковок необходимая информация также будет переноситься в консоль). По окночании установки, если все необходимые условия будут приемлемы, программа завершит установку и доложит об этом в консоли. Для выхода из программы достаточно закрыть окно с консолью, либо нажать <Enter> с пустым полем для ввода.

Для удаления пакетов данных используется программа <DeleteDataPack.exe>. Если путь <data\DataPacks> отсутствует, то при запуске предложенной программы та создаст нужную папку. Все пакеты данных для удаления (установленные) начинаются либо с <#dp_>, либо с <#datapack_> и загружаются в папку <DataPacks>.
В случае запуска программы с имеющейся папкой, та начнет поиск пакетов данных и предложит удалить имеющиеся (в случае отсутствия пакетов программа предложит завершить работу). При выборе пакета для удаления программа сначала покажет его содержимое, а затем уже предложит его удалить (в случае отказа от удаления Вы вернетесь к списку пакетов). При соглашении на удаление пакета программа начнет его обрабатывать (обработка обычно занимает менее секунды, но это время зависит от 'мощности' Вашего устройства). В случае возникновения проблем необходимая информация будет переноситься в консоль. По окночании удаления, если все необходимые условия будут приемлемы, программа завершит удаление и доложит об этом в консоли [Примечание: Ксли какие-либо объекты, указанные в установленном пакете данных, отсутствуют - программа не выдаст ошибку, а просто пропустит этот момент, т.е. если вы попытаетесь удалить пакет данных, который до этого не был установлен, ничего страшного не будет]. Для выхода из программы достаточно закрыть окно с консолью, либо нажать <Enter> с пустым полем для ввода.

Ниже представлена инструкция для создания пакетов данных:
1) Перейдите по пути <data\DataPacks>
2) Создайте новый текстовый файл формата <.txt> и задайте ему имя по следующему плану:
-> Выберите начальную приставку для имени файла: <dp_> или <datapack_>
-> Введите любое имя для файла (рекомендуется выбирать говорящее имя, которое будет описывать цель для пакета данных).
-- Примеры имен пакетов данных: <dp_circle> (Создание фигуры 'круг'), <datapack_square> (Фигура 'квадрат'), <dp_star> (Фигура 'звезда'), <datapack_house> (Фигура 'дом').
-> Завершите редактирование имени файла [Примечание: Имя для пакета данных никак не влияет на его работу, т.е. Вы можете выбрать АБСОЛЮТНО любое имя, главное, чтобы не было конфилкта между прочими пакетами]
3) Откройте только что созданный Вами файл и запишите параметры из предложенного списка:
-> # info # (Информация) [Необязательный] - Создает описание пакета
-> # type # (Тип) - Задает тип пакета [На данный момент есть один тип: figure], а также параметры к этому типу
-- Список параметров [Примечание: в квадратных скобках ('[' и ']') записаны обязательные параметры, а в круглых ('(' и ')') - необязательные]:
--> figure: [Имя], [Клавиша], (Аргументы и их значения)
-> # code # (Код) - Задает код для написания в файл <figures.py>
-- Пример написания параметров на примере пакета данных для фигуры 'круг':
--> # info #
--> Круг
--> # type #
--> figure circle 1 radius 20
--> # code #
--> def figure_circle(x):
--> 	circle(x['radius'])
-- [Примечание: Необязательно соблюдать все разделения, которые не являются разделениями слов, в виде пробелов и разрывов строки. Вот код выше, но уже без всех необязательных разделений:
-- <#info#Круг#type#figure circle 1 radius 20#code#def figure_circle(x): circle(x['radius'])>]
